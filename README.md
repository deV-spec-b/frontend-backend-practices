# 🛒 Интернет-магазин товаров

**Полноценное веб-приложение с авторизацией, ролевой моделью (RBAC) и админ-панелью**

---

## 📚 Стек технологий

### Backend
| Технология | Назначение |
|------------|------------|
| **Express** | Серверная часть |
| **JWT** | Access + Refresh токены |
| **bcrypt** | Хеширование паролей |
| **Swagger** | Документация API |
| **nanoid** | Генерация уникальных ID |
| **cors** | Кросс-доменные запросы |

### Frontend
| Технология | Назначение |
|------------|------------|
| **React** | Клиентская часть |
| **React Router** | Маршрутизация |
| **Axios** | HTTP-запросы с интерсепторами |
| **SCSS** | Стилизация |

## 🎯 Функциональность

### 🔐 Аутентификация
- ✅ Регистрация с выбором роли (user / seller / admin)
- ✅ Вход с выдачей пары токенов (access + refresh)
- ✅ Автообновление access-токена при 401 ошибке

- ✅ Редактирование товара (seller / admin)

### 👥 Управление пользователями (только admin)
- ✅ Список всех пользователей
- ✅ Редактирование имени, фамилии, email
- ✅ Изменение роли (user → seller → admin)

## 🎭 Роли и права доступа (RBAC)

| Роль | Доступ |
|------|--------|
| **Гость** (не авторизован) | Регистрация, вход |
| **Пользователь** | Всё, что гость + просмотр профиля, список товаров |
| **Продавец** | Всё, что пользователь + создание и редактирование товаров |
| **Администратор** | Полный доступ + управление пользователями |

### Таблица доступа к маршрутам

| Маршрут | Метод | Доступ |
|---------|-------|--------|
| `/api/auth/register` | POST | Гость |
| `/api/auth/login` | POST | Гость |
| `/api/auth/refresh` | POST | Гость |
| `/api/auth/me` | GET | Пользователь+ |
| `/api/users` | GET | Администратор |
| `/api/users/:id` | GET/PUT/DELETE | Администратор |
| `/api/products` | GET | Пользователь+ |
| `/api/products` | POST | Продавец+ |
| `/api/products/:id` | GET | Пользователь+ |
| `/api/products/:id` | PUT | Продавец+ |
| `/api/products/:id` | DELETE | Администратор |

## Практика 17: Напоминания с датой/временем

### Реализовано:
- ✅ Добавление поля `datetime-local` в форму для выбора даты/времени
- ✅ Хранение заметок с уникальным `id` и полем `reminder` (timestamp)
- ✅ Планирование push-уведомлений на сервере через `setTimeout`
- ✅ Хранилище активных напоминаний (`reminders = new Map()`)
- ✅ Кнопка **"Отложить на 5 минут"** в push-уведомлении
- ✅ Эндпоинт `/snooze` для откладывания напоминания
- ✅ Обработчик `notificationclick` в Service Worker

### Процесс работы напоминания:
1. Пользователь выбирает дату/время и текст
2. Клиент отправляет событие `newReminder` на сервер
3. Сервер вычисляет задержку и устанавливает `setTimeout`
4. По истечении времени сервер отправляет push-уведомление
5. В уведомлении есть кнопка **"Отложить на 5 минут"**
6. При нажатии отправляется запрос на `/snooze`, таймер обновляется
