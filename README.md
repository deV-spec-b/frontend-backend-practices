# 🛒 Интернет-магазин товаров

**Полноценное веб-приложение с авторизацией, ролевой моделью (RBAC) и админ-панелью**

---

## 📚 Стек технологий

### Backend
| Технология | Назначение |
|------------|------------|
| **Express** | Серверная часть |
| **JWT** | Access + Refresh токены |
| **bcrypt** | Хеширование паролей |
| **Swagger** | Документация API |
| **nanoid** | Генерация уникальных ID |
| **cors** | Кросс-доменные запросы |

### Frontend
| Технология | Назначение |
|------------|------------|
| **React** | Клиентская часть |
| **React Router** | Маршрутизация |
| **Axios** | HTTP-запросы с интерсепторами |
| **SCSS** | Стилизация |

---

## 🎯 Функциональность

### 🔐 Аутентификация
- ✅ Регистрация с выбором роли (user / seller / admin)
- ✅ Вход с выдачей пары токенов (access + refresh)
- ✅ Автообновление access-токена при 401 ошибке
- ✅ Защищённые маршруты (редирект на `/login`)

### 🛒 Товары
- ✅ Просмотр списка всех товаров (миксеры, блендеры, кофемашины и т.д.)
- ✅ Просмотр детальной информации по id
- ✅ Создание товара (seller / admin)
- ✅ Редактирование товара (seller / admin)
- ✅ Удаление товара (admin)

### 👥 Управление пользователями (только admin)
- ✅ Список всех пользователей
- ✅ Редактирование имени, фамилии, email
- ✅ Изменение роли (user → seller → admin)
- ✅ Блокировка пользователя

---

## 🎭 Роли и права доступа (RBAC)

| Роль | Доступ |
|------|--------|
| **Гость** (не авторизован) | Регистрация, вход |
| **Пользователь** | Всё, что гость + просмотр профиля, список товаров |
| **Продавец** | Всё, что пользователь + создание и редактирование товаров |
| **Администратор** | Полный доступ + управление пользователями |

### Таблица доступа к маршрутам

| Маршрут | Метод | Доступ |
|---------|-------|--------|
| `/api/auth/register` | POST | Гость |
| `/api/auth/login` | POST | Гость |
| `/api/auth/refresh` | POST | Гость |
| `/api/auth/me` | GET | Пользователь+ |
| `/api/users` | GET | Администратор |
| `/api/users/:id` | GET/PUT/DELETE | Администратор |
| `/api/products` | GET | Пользователь+ |
| `/api/products` | POST | Продавец+ |
| `/api/products/:id` | GET | Пользователь+ |
| `/api/products/:id` | PUT | Продавец+ |
| `/api/products/:id` | DELETE | Администратор |

## Практика 7: Базовая аутентификация (bcrypt)

### Реализовано:
- ✅ Регистрация пользователей с хешированием паролей (bcrypt, соль)
- ✅ Вход с проверкой пароля
- ✅ Swagger-документация для всех маршрутов
- ✅ Пользователи хранятся в памяти (массив users)

### Технологии:
- `bcrypt` — хеширование паролей

## Практика 8: JWT-токены

### Реализовано:
- ✅ Генерация JWT-токена при входе (accessToken)
- ✅ Middleware `authMiddleware` для проверки токена
- ✅ Защищённый маршрут `GET /api/auth/me` (данные текущего пользователя)
- ✅ Библиотека `jsonwebtoken`



## Практика 9: Refresh-токены

### Реализовано:
- ✅ Разделение на access-токен (15 минут) и refresh-токен (7 дней)
- ✅ Хранилище валидных refresh-токенов (`Set`)
- ✅ Маршрут `POST /api/auth/refresh` для получения новой пары токенов
- ✅ Ротация refresh-токенов (старый удаляется, новый создаётся)

## Практика 11: RBAC (ролевая модель)

### Реализовано:

#### Роли:
 Роль  Права 

 **Гость** (не авторизован) | Только регистрация и вход 
 **Пользователь** | Просмотр товаров, просмотр профиля 
 **Продавец** | Просмотр товаров + создание и редактирование товаров 
 **Администратор** | Все права продавца + управление пользователями 

#### Маршруты с учётом ролей:

 Маршрут  Метод  Доступ  Описание 
 `/api/auth/register`  POST  Гость  Регистрация 
 `/api/auth/login`  POST  Гость  Вход 
 `/api/auth/refresh`  POST  Гость  Обновление токенов 
 `/api/auth/me`  GET  Пользователь+  Данные текущего пользователя 
 `/api/users`  GET  Администратор  Список всех пользователей 
 `/api/users/:id`  GET  Администратор  Получить пользователя по id 
 `/api/users/:id`  PUT  Администратор  Обновить информацию пользователя 
 `/api/users/:id`  DELETE  Администратор  Заблокировать пользователя 
 `/api/products`  POST  Продавец+  Создать товар 
 `/api/products`  GET  Пользователь+  Получить список товаров 
 `/api/products/:id`  GET  Пользователь+  Получить товар по id 
 `/api/products/:id`  PUT  Продавец+  Обновить товар 
 `/api/products/:id`  DELETE  Администратор  Удалить товар
